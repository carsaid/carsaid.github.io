表-Kubernetes常用命令

# 表-Kubernetes常用命令

# 思维导图

为了方便阅读和记忆，我正在整理 Kubernetes 的常用命令，并将其做成一个思维导图，敬请期待。

下述命令表格停止更新，之后会全部替换成思维导图。

# 1. 集群信息查询

# 综合信息

序号 命令 说明
1 kubectl cluster-info 查看集群的基本信息。
2 kubectl cluster-info dump 转储 “适合调试和诊断群集问题” 的集群完整信息，默认将所有信息转储到标准输出（注：信息量比较大）。
3 kubectl cluster-info dump --output-directory <目录名称> 在指定的目录中构建一组文件，文件中包含集群的转储信息。

提示

命令kubectl cluster-info dump默认情况下只转储命名空间kube-system中的内容，但是你可以使用--namespaces标志来切换到不同的命名空间，或者指定--all-namespaces来转储所有的命名空间。

# 版本信息

序号 命令 说明
1 kubectl version 查看 K8s 版本信息。
2 kubectl version --short 查看精简的 K8s 版本信息（注：--short选项已被弃用，在未来的版本中将被移除。--short的输出将成为kubectl version的默认值）。

# api-version

序号 命令 说明
1 kubectl api-versions 查看当前 K8s 版本支持的 api-version

# 2. 节点操作

序号 命令 作用
1 kubectl get nodes 查看当前集群内的所有节点
2 kubectl top nodes 查看当前集群内的所有节点工作负载（需要提前安装监控类插件）
3 kubectl cordon <节点名称> 停止节点——停止对该 node 的调度，node 状态变为SchedulingDisabled，后续新创建的 Pod 将不会在该 node 上运行。
该 node 上原有的 Pod 不受影响，会继续运行。
4 kubectl drain <节点名称> 驱逐节点——停止对该 node 的调度，node 状态变为SchedulingDisabled，并清空上面的所有 Pod，被清空的 Pod 会被调度到其他 node 上运行。
5 kubectl delete nodes <节点名称> 删除节点——从 master 上删除该 node，失去对其的控制权（注：被删除的节点无法恢复！）。
6 kubectl uncordon <节点名称> 恢复节点——将一个状态为SchedulingDisabled的 node 恢复为可用状态。
（注：被kubectl delete删除的节点无法恢复！）
序号 可选标志 作用
1 --force 强制进行当前操作。
2 --delete-local-data 删除节点本地数据——即使该节点上的某个 Pod 使用 emptyDir 来存储数据（节点丢失时将被删除的本地数据），也继续进行删除操作。
3 --ignore-daemonsets 忽略由 DaemonSet 管理的 Pod。

# 3. Pod相关

# Pod信息

序号 命令 作用
1 kubectl get pods 查看默认命名空间内的 Pod
2 kubectl top pods 查看默认命名空间内的 Pod 工作负载（需要提前安装监控类插件）
序号 可选标志 作用
1 -n <命名空间名称> 查看指定命名空间内的 Pod
2 --all-namespaces 查看所有命名空间内的 Pod
3 -A 同上，--all-namespaces 的简写

# kubeadm

# kubeadm token

序号 命令 作用
1 kubeadm token create --print-join-command 创建新的 token 并打印加入集群的命令，其他主机可以通过该命令 来加入集群。

# ctr命令行工具

# ctr images

提示：images 可以缩写为 image 或 i。

序号 命令 作用
1 ctr i ls 查看默认命名空间中的镜像
2 ctr -n <命名空间名称> i ls 查看指定命名空间中的镜像
3 ctr i pull <镜像地址/名称> 拉取镜像到默认的命名空间中
4 ctr -n <命名空间名称> i pull <镜像地址/名称> 拉取镜像到指定的命名空间中

# ctr namespaces

提示：namespaces 可以缩写为 namespace 或 ns。

序号 命令 作用
1 ctr ns ls 查看所有的镜像命名空间
2 ctr ns create <名称> 创建镜像命名空间
